# ポートフォリオサイト

Cloudflare Pages対応の静的ポートフォリオサイトです。

## 🚀 特徴

- **静的サイト**: HTML/CSS/JavaScriptのみで構成
- **レスポンシブ対応**: スマートフォンからデスクトップまで最適化
- **モダンなデザイン**: グラデーションとアニメーションを使用
- **Cloudflare Pages対応**: そのままデプロイ可能
- **アクセシビリティ対応**: キーボードナビゲーションとスクリーンリーダー対応

## 📁 フォルダ構成

```
portfolio-site/
├── index.html              # ホームページ
├── qualifications.html     # 取得資格ページ
├── works.html             # 制作作品一覧ページ
├── works/                 # 作品詳細ページ
│   ├── app1.html          # タスク管理アプリ詳細
│   └── app2.html          # ECサイト詳細
├── css/
│   └── style.css          # 全スタイルシート
├── js/
│   └── script.js          # JavaScript機能
├── images/                # 画像ファイル
│   └── README.md          # 画像説明
└── README.md              # このファイル
```

## 🛠 技術スタック

- **HTML5**: セマンティックマークアップ
- **CSS3**: Flexbox, Grid, アニメーション
- **Vanilla JavaScript**: フレームワークなし
- **レスポンシブデザイン**: モバイルファースト

## 🚀 Cloudflare Pagesへのデプロイ

1. このリポジトリをGitHubにプッシュ
2. Cloudflare Pagesダッシュボードで新しいプロジェクトを作成
3. GitHubリポジトリを接続
4. ビルド設定:
   - フレームワーク: なし
   - ビルドコマンド: なし
   - 出力ディレクトリ: `.`

## 📝 カスタマイズ方法

### 基本情報の変更
- `index.html`の名前や自己紹介を編集
- `qualifications.html`の資格情報を更新
- `works.html`の作品情報を変更

### スタイルの変更
- `css/style.css`のカラーテーマを変更
- フォントやレイアウトを調整

### 画像の追加
- `images/`フォルダに画像を配置
- HTML内の画像パスを更新

### 新しい作品ページの追加
1. `works/`フォルダに新しいHTMLファイルを作成
2. `works.html`にカードを追加
3. 必要に応じてCSSやJavaScriptを調整
